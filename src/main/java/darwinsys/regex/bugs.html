Regular Expressions API: Implementation Bugs and TODO

This is a tiny litle NFA-style regex implementation, done by Ian darwin many years ago as a proof-of-concept and never quite finished.

Caveats

The current implementation is very slow. The java.util.regex package is several times faster, partly because it compiles to (its own) byte code.

Bugs

The following bugs are present in this version:

  • backtracking is not yet implemented, so a .* followed by something may (i.e., will usually) eat up the something.

    The following functionality is documented but not implemented; must be added real soon now.

    • Ignore-case (fix in SE.amatch() arg list, SEchar, SEccl)

    Future work

    The following are planned to be implemented Real Soon Now(tm):

    • Use Match match() to write sub() functionality.

    The following are possible additions.

    • \f numeric digits plus . e f (\f+ for floating number)
    • \N any end-of-line character (short for [\r\n])
